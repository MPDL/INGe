Installation Prerequisites

Introduction

eSciDoc is a complex software stack, consisting of various software components. This installer will guide you step by step through the installation process. However, it may be necessary to install and configure a runtime environment prior to running this installer. The runtime environment consists of two components:

  • Java SDK
  • Relational Database

Please make sure both software components are properly installed onto the machine you will install the eSciDoc Infrastructure.

Java JDK

Running the eSciDoc Infrastructure requires a Java Software Development Kit (JDK). A run-time environment (JRE) is sufficient to run this installer, but starting the system after a successful installation will fail. Please make sure that you have a JDK Version 1.6 or newer installed on your system.

You can download the latest JDK software at no cost from http://java.sun.com/javase/downloads/.

Relational Database Management System

The eSciDoc Infrastructure currently supports only PostgreSQL as open source database management system.

  • PostgreSQL 8.4 or newer

The database needs to be up and running during the installation process. You will need to provide credentials for a database user with sufficient privileges to create login roles and databases. This user will only be used to create two login roles (typically "fedoraAdmin" and "escidoc") and three databases ("fedora3", "escidoc-core", "riTriples").

If you want to use an existing instance of PostgreSQL (which may reside on a remote server), please make sure that it meets the following requirements:

PostgreSQL

Software version:≥ 8.4
Database encoding:UTF-8
